# Retrospective Guide

Forge v2 retrospectives are milestone retrospectives, not generic “what happened” summaries.

## Inputs to inspect

- `docs/documentation.md`
- `.forge/state/current/queue.json`
- `.forge/state/current/last_validation.json`
- `git log`
- `docs/backlog.md`

## Questions that matter

- Did the milestone stay inside scope?
- Did validation reflect the actual user surface?
- Where did the agent waste time or oscillate?
- What durable rule should move into docs or AGENTS?

## Useful outputs

- sharper milestone sizing for the next run
- better validation commands or smoke scenarios
- clearer durable docs
- backlog items tagged with the right context

## Anti-patterns

- vague praise or blame with no concrete follow-up
- changing product scope in retro
- skipping validation evidence
- forgetting to archive the current milestone snapshot
